I continue to watch the development of ASP.NET and PHP 5.0 as competing web platforms. I know that most really consider Java to be the real contender, but the rapid growth of PHP makes it a new contender :)

I found some [great commentary](<https://web.archive.org/web/20060104214356/http://blogs.ittoolbox.com/c/programming/archives/005608.asp?rss=1>) on the new ASP.NET release written by [George Alexander.](<https://web.archive.org/web/20050805013750/http://blogs.ittoolbox.com/c/programming//authorbio.asp>)

> There's plenty of new stuff in ASP.Net 2.0. More data controls (XHTML server controls, Login Controls, Web Parts, tons of extra controls), more new services and APIs (Membership, Role Management, Profiles, Site Maps) and a couple of new Page Framework stuff including some UI stuff like page inheritance and themes and skins and web part development and the ability to use different programming languages within a project on different files. All this new stuff sounds super cool - with all new controls and APIs coming into play and a SQL Server 2005 thats CLR compliant, microsoft probably programming just got better. 

Now we just need a legitimate comparison between Java, .NET, and PHP for web application development.
