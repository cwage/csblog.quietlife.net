**Where did Ruby come from?[](<http://www.oddee.com/item_96451.aspx>)**

Like many [cool items](<http://www.oddee.com/item_96451.aspx>) in Japan, in the early 1990's, a Japanese developer named Yukihiro Matsumoto wrote the Ruby language to try to help every programmer become more productive and happy. In 2003, BaseCamp's David Heinemeier Hansson created Ruby on Rails to help develop a website much faster in a simple way.

**So what exactly is Ruby on Rails?**

In short, developers use Ruby on Rails as a platform to build dynamic websites, web applications and web services. It can be said that Ruby on Rails helps make common development tasks easier for basic websites and thrives on the saying of "don't repeat yourself". It lets you write less code by favoring convention over configuration. Who wouldn't like that?

**What does Rails mean?**

I once read "Rails is much more than just a tube of glue with which to stick Ruby on the web." Rails is a web application development framework written in the Ruby language. A framework is basically a frame that you can begin building upon to make a web application. What exactly is a web application you may ask? (Remember, this is for design dummies like myself which is why I'm writing these questions). A web application is more than just a webpage. A web application is more like a normal application on your computer except the difference is that it uses your web brewer as its base. Gmail is a really great example of a strong web application.

**'Ruby on Rails' Hall of Fame:**

Twitter, Basecamp, Hulu, Urban Dictionary, XING and Yellowpages.com to name a few.

**Pros to using Ruby on Rails?**

1\. What used to take a programmer months to complete, can now hopefully be done within a few days.  
2\. Hundreds and hundreds of available plug-ins  
3\. Less Coding- RoR requires less amount of code to implement the business logic.  
4\. Its free to be implemented by everyone
