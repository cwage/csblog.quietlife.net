While Open Source solutions have their appeal, the ultimate test of their worth is determined by their ability to get the required job done. In the case of corporate communication, we've felt that there is no good solution for the Open Source community.

We've tried to use solutions like Communigate, but they are poor substitutes for Exchange. Moreover, we've had serious problems having an integrated PIM that easily sync to Open Source solutions.

For example, we have staff members with the Palm Treo 650 - a fantastic phone with many 'smart' features. Unfortunately, we've had very little success integrating that phone with any of our open source solutions. The result - we must use Exchange or a local Outlook installation to have it work.

Perhaps that problem has a solution (or at least a possible one). Chris found an open source project that is attempting to address this problem. [Sync4j ](<http://sync4j.funambol.com/main.jsp?main=theproject>)is a mobile application server, and more. It includes a suite of tools to develop, deploy and manage any mobile project.

The home page states:

> Sync4j consists of:
> 
> * Sync4j Server: a mobile application server with connectors to SQL relational databases, Microsoft Exchange, Lotus Domino and SugarCRM.  
>  * Sync4j Clients: applications for Outlook, Windows Mobile PocketPC, BlackBerry, Palm and iPod that you can use to synchronize your PIM data (address book and calendar) with the server  
>  * Sync4j Email Gateway: a POP, IMAP and SMTP gateway, to support your mobile email needs.  
>  * Sync4j Software Development Kit: tools to develop sometimes-connected mobile applications on devices (in Java - J2SE and J2ME - and C++) and to add data sources to the server  
>  * Sync4j Device Management: an OMA DM server to remotely manage mobile devices 

If you're pressed for time, you can click here to [view the picture](<http://sync4j.funambol.com/imgs/overview.jpg>) of what Sync4j is trying to accomplish.
